Basándote en el tutorial de Max Techera, aquí tienes la recopilación de todos los comandos de terminal (y comandos slash) junto con el sistema de Hooks, organizados para maximizar la seguridad y la eficiencia en el desarrollo: \#\#\# Comandos de Terminal y Comandos Slash 1\. **claude**: Inicia el REPL (instancia local interactiva) de Claude Code aislado en la carpeta de tu proyecto 1\. 2\. **claude \--dangerously-skip-permissions**: Inicia Claude en modo "Auto", omitiendo las confirmaciones de permisos de comandos. (Se recomienda usar solo si se tiene configurado un buen sistema de ramas en Git por seguridad) 2, 3\. 3\. **/permissions**: Permite configurar qué comandos específicos puede ejecutar Claude de forma automática sin pedir tu aprobación 2\. 4\. **/output-style**: Ajusta cómo Claude formatea sus respuestas o le pide que explique su razonamiento antes de actuar 4\. 5\. **/init**: Escanea tu proyecto y genera el archivo de memoria persistente CLAUDE.md para que el agente recuerde tu arquitectura, convenciones y reglas en futuras sesiones 5\. 6\. **/clear**: Limpia el contexto del chat activo. Fundamental para la eficiencia, evitando que Claude mezcle código o se confunda cuando cambias de funcionalidad 6, 7\. 7\. **/resume**: Permite volver a un chat o sesión de trabajo anterior 7\. 8\. **/compact**: Resume y compacta el contexto activo conservando solo las instrucciones que especifiques, ideal cuando te estás quedando sin tokens de contexto 7\. 9\. **/agents**: Crea, define y configura sub-agentes especializados (por ejemplo, para testeo o revisión de código) para no saturar la ventana de contexto principal 8\. 10\. **Comandos de Git Worktrees (git worktree add  y cd && claude)**: Utilizados para instanciar múltiples ramas en carpetas separadas. Permite ejecutar múltiples sesiones de Claude Code en paralelo sin conflictos de código 9, 10\. 11\. **/hooks**: Inicia la interfaz de configuración para gestionar las automatizaciones (Hooks) del proyecto 11\. 12\. **Comandos custom (ej. /review)**: Comandos personalizados por el usuario guardados en .claude/commands/ para ejecutar flujos de trabajo repetitivos y compartirlos con el equipo 12, 13\. \--- \#\#\# Sistema de Hooks (Automatizaciones) Los Hooks se ejecutan automáticamente durante el ciclo de vida de Claude. El tutorial presenta implementaciones específicas pensadas para blindar el código (**seguridad**) y eliminar tareas manuales (**eficiencia**): 1\. **Auto-formatear código después de cada edit (Tipo: PostToolUse)** \* **Propósito:** Ejecuta automáticamente un formateador (como Prettier) cada vez que Claude edita un archivo. \* **Objetivo:** Mejora drásticamente la **eficiencia** al mantener el código consistente sin necesidad de intervención manual 14\. 2\. **Logging de todos los comandos bash (Tipo: PostToolUse)** \* **Propósito:** Crea un registro completo de todos los comandos de terminal que Claude ejecuta. \* **Objetivo:** Aumenta tanto la **seguridad** (para compliance y auditoría de qué hizo el agente) como la **eficiencia** a la hora de realizar *debugging* si algo falla 14\. 3\. **Bloquear edits en archivos de producción (Tipo: PreToolUse)** \* **Propósito:** Intercepta a Claude antes de que use la herramienta de edición, denegando modificaciones si la ruta coincide con archivos críticos (ej. carpetas /config/ de producción). \* **Objetivo:** Maximiza la **seguridad** garantizando que la IA no pueda tocar ni romper infraestructura o configuraciones críticas bajo ninguna circunstancia 14, 15\. 4\. **Validar que no haya API keys en el prompt (Tipo: UserPromptSubmit)** \* **Propósito:** Escanea los mensajes que el usuario envía a Claude antes de procesarlos. \* **Objetivo:** Proporciona una red de **seguridad** crítica, previniendo la fuga accidental de credenciales, secretos o tokens hacia la IA 15\. 5\. **Validar que los tests pasen antes de hacer commit (Tipo: PreToolUse)** \* **Propósito:** Bloquea la acción de hacer un *commit* en Git si la suite de pruebas automatizadas falla. \* **Objetivo:** Aumenta la **seguridad** de los despliegues evitando que código roto llegue a producción, y fomenta la **eficiencia** erradicando la mala práctica de "arreglar los tests más tarde" 16\. 6\. **Notificación custom cuando Claude espera input (Tipo: Notification)** \* **Propósito:** Dispara una notificación nativa del sistema operativo en tu computadora cuando Claude necesita confirmación para un permiso o ha terminado una tarea larga. \* **Objetivo:** Dispara la **eficiencia** del desarrollador, permitiéndote cambiar de ventana y trabajar en otras cosas sabiendo que el sistema te avisará exactamente cuando requiera tu atención 16\.
