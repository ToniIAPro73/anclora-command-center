# Prompt de normalización final por modo para Google Stitch - Canonical light y dark

Este recurso recoge los prompts finales de ajuste para `canonical dark` y `canonical light`. No están pensados para abrir nuevas direcciones visuales, sino para cerrar incoherencias y normalizar los dos modos como una única familia premium Anclora.

## Cuándo usarlo

- cuando ya existe una dirección `canonical`
- cuando no hace falta más exploración, sino refinamiento
- cuando el objetivo es corregir contradicciones de sistema, modularidad o tono
- cuando se quiere igualar la fuerza relativa entre `light` y `dark`

## Prompt final para canonical dark

```text
Refine the current Anclora canonical dark landing with a normalization pass, not a redesign.

The core direction is already correct. Keep the existing atmospheric depth, editorial authority, tonal richness, asymmetry, and premium dark identity. Do not branch into a new concept. Do not create a different visual language. Improve only what is still inconsistent.

Main goals:
- preserve the dark mode as the strongest Anclora premium identity
- reduce the overly financial/advisory tone so the system feels reusable across AI, editorial, data, platform, and premium service products
- eliminate visual contradictions between the design rationale and the actual layout
- make the dark system feel more canonical, modular, and reusable

Adjustments required:

1. SYSTEM CONSISTENCY
- Remove visible divider-line logic where the system claims tonal layering.
- Replace borders, separator rows, and grid-divider effects with spacing, surface shifts, depth, and compositional breaks.
- Ensure asymmetry is intentional and structurally meaningful, not decorative.

2. REUSABILITY
- Reduce language and visual cues that make the page feel too specific to financial advisory.
- Keep the premium strategic tone, but make modules feel adaptable to multiple Anclora product types.
- Make hero, proof, narrative, FAQ, CTA, and footer feel like reusable ecosystem blocks.

3. COMPONENT NORMALIZATION
- Refine CTA hierarchy so there is one dominant action and cleaner supporting actions.
- Make trust/proof sections feel more authored and less like standard logo rows.
- Make FAQ feel premium and system-native, not a styled default pattern.
- Make footer feel canonical and reusable for multiple product families.

4. DARK MODE QUALITY
- Keep the deep teal, navy, warm near-black, muted gold atmosphere.
- Preserve calm readability and premium contrast.
- Avoid making the dark mode harsher, brighter, or more “cyber”.
- Keep it elegant, quiet, editorial, and architectural.

5. MOBILE
- Preserve focal points on mobile.
- Tighten spacing where the layout becomes too passive.
- Keep the premium rhythm and hierarchy intact on narrow screens.

Do not:
- invent a new concept
- revert to generic SaaS patterns
- add decorative noise
- overcomplicate the design
- overemphasize finance-only storytelling

Deliver:
- a cleaner, more canonical dark version
- better consistency with the stated design principles
- more reusable modular blocks
- less domain-specific bias
- stronger Anclora ecosystem applicability
```

## Prompt final para canonical light

```text
Refine the current Anclora canonical light landing with a normalization pass, not a redesign.

The light version already has clarity, elegance, and premium restraint. Keep that. Do not create a new visual direction. Do not start over. The task is to make the light mode feel more canonical, more compositionally confident, and more clearly part of the same family as the dark mode.

Main goals:
- preserve the light mode’s openness, legibility, warmth, and editorial calm
- make it feel more distinctive and less like a premium consulting template
- increase compositional authority so it matches the strength of the canonical dark version
- strengthen family resemblance between light and dark
- make the system more reusable across AI, editorial, data, platform, and premium service apps

Adjustments required:

1. FAMILY CONSISTENCY
- Bring the light version closer to the compositional authority and system logic of the dark version.
- Ensure light and dark clearly belong to one Anclora premium family.
- Match hierarchy, cadence, module logic, and premium tone across both modes.

2. VISUAL DISCIPLINE
- Remove visible divider-line logic and standard grid-divider patterns where tonal layering should do the work.
- Replace generic card-grid feeling with more editorial structure, layered surfaces, and stronger rhythm.
- Increase intentional asymmetry in selected sections without making the layout unstable.

3. LIGHT MODE CHARACTER
- Preserve the warm editorial neutrals and premium calm.
- Avoid flatness, safe corporate polish, or generic premium-consulting aesthetics.
- Add more tension, framing, and visual authorship so the page feels more memorable and canonical.

4. REUSABILITY
- Reduce domain-specific cues that over-anchor the page in advisory positioning.
- Make modules feel reusable for AI, editorial, data, platform, and service products.
- Clarify canonical block roles: hero, value proposition, narrative, proof, FAQ, CTA, footer.

5. COMPONENT NORMALIZATION
- Improve CTA presence and hierarchy.
- Make proof/trust feel more authored and less standard.
- Make FAQ feel designed, not default.
- Make footer feel more like a canonical ecosystem footer than a premium website footer.

6. MOBILE
- Tighten vertical rhythm.
- Preserve stronger focal moments on small screens.
- Ensure the page still feels premium and intentional in mobile, not just spacious.

Do not:
- create a new aesthetic branch
- flatten the design into neutrality
- reintroduce generic SaaS composition
- overdecorate the page
- make the light mode feel weaker than dark mode

Deliver:
- a more canonical light version
- stronger family consistency with dark mode
- less template feeling
- more editorial authority
- clearer reusable system blocks
- stronger Anclora premium identity
```
