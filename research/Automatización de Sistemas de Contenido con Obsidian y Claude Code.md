**Flujo de trabajo secuencial para automatizar un sistema de contenido**

1. **Estructuración del Vault para accesibilidad de la IA:** El primer paso es establecer una jerarquía de carpetas lógica en Obsidian, típicamente usando tres carpetas principales: *01\_Inspiration* (para referencias y desgloses de IA), *02\_Production* (para guiones activos y storyboards) y *03\_Systems* (para guías de estilo y bibliotecas de prompts) 1\.  
2. **Conexión e indexación con Claude Code:** Se ejecuta Claude Code localmente sobre el directorio de Obsidian para integrar la capa autónoma de IA a tu bóveda 1\.  
3. **Storyboarding automatizado:** Una vez que el guion está escrito en un archivo markdown, se le da a la IA el comando de transformar ese texto en una tabla de guion gráfico (storyboard) 2\. Esta tabla incluye columnas para la toma visual, la mecánica técnica (ej. ajustes de iluminación RGB) y el ritmo emocional, asegurando que cada fracción de segundo esté planeada antes de grabar 2\.  
4. **Estructuración narrativa de 5 actos:** Se programa a Claude Code para que imponga un arco narrativo de 5 actos en cada guion del vault (gancho, conflicto, construcción, resolución y llamado a la acción), lo que asegura una narración técnica emocionalmente resonante y un camino psicológico probado hacia la viralidad 3, 4\.

**Cómo usar Claude Code para indexar el vault**Al operar Claude Code localmente sobre el directorio de Obsidian, la herramienta no solo ve archivos sueltos, sino que **lee, indexa y puede reestructurar toda tu bóveda** 5\. Puedes ejecutar comandos directos como: *"Analiza mis últimos cinco guiones virales en la carpeta 'Production' y genera una plantilla de storyboard para mi próximo video"* 1\. Claude Code indexará y analizará tu **estilo de escritura específico** desde esos archivos para garantizar que las nuevas plantillas y resultados se sientan auténticos para tu marca, sin hacer suposiciones genéricas 1\.  
**Mantener la consistencia de marca con una 'Style Guide'**Para asegurar que tu contenido se vea y suene igual incluso al escalar la producción, debes crear una **biblioteca de prompts de 'Style Guide'** (Guía de Estilo) dentro de la carpeta *Systems* de Obsidian 1, 6\. Esta guía debe definir **parámetros estrictos de Tipografía** (fuentes y reglas para los subtítulos), **Estética del B-Roll** (tipos de tomas y efectos visuales) y **Textura de Audio** (BPM y filtros de ecualización) 7\. Luego, alimentas esta guía en Claude Code o Cursor y le pides a la IA que evalúe tus guiones; la IA marcará automáticamente cualquier línea o elemento que no encaje con la "vibra" de tu marca, garantizando consistencia independientemente de quién edite el video 3\.  
**Reestructuraciones masivas de metadatos ("Growth Ops")**A medida que la marca crece, el volumen de archivos y metadatos se vuelve abrumador. En lugar de organizar manualmente, utilizas Cursor y Claude Code para realizar reestructuraciones masivas 8\. Simplemente apuntas Claude Code a la raíz de tu proyecto y ejecutas un comando como: *"Reorganiza todos los proyectos del Q1 2026 a la nueva nomenclatura 'basada en series' y actualiza los metadatos en el calendario de contenido maestro"* 8\. Esta automatización extrema **te permite dedicar el 90% de tu tiempo a la creación creativa y solo el 10% a la administración del sistema** 8\.
