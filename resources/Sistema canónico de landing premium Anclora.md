# Sistema canónico de landing premium Anclora

Esta nota fija la decisión final tras la exploración en `Google Stitch` para el esqueleto de landing premium reusable del ecosistema Anclora.

La fase de exploración se considera cerrada. A partir de este punto, el criterio ya no es seguir buscando nuevas direcciones, sino reutilizar y adaptar este sistema base para productos concretos.

## Decisión final

El sistema canónico queda definido por una pareja de modos complementarios:

- `dark` como referencia principal de identidad
- `light` como traducción operativa y accesible de la misma familia

La versión `dark` se adopta como el eje de marca porque expresa mejor la autoridad editorial, la profundidad tonal, la modularidad y la singularidad del sistema premium Anclora.

La versión `light` se conserva como contraparte válida del mismo lenguaje, útil cuando el contexto pida más apertura visual, legibilidad ambiental o una atmósfera menos densa.

## Qué se considera canónico

### Identidad general

- dirección editorial premium
- `quiet luxury`
- precisión técnica sin frialdad
- composición respirada
- contraste entre serif con personalidad y sans funcional
- motion sobrio y contenido
- sensación de sistema diseñado, no de plantilla SaaS

### Regla de jerarquía

- el primer viewport debe explicar rápido qué es el producto y cuál es la siguiente acción
- debe existir un único CTA dominante por viewport principal
- la landing debe priorizar ritmo, foco y estructura de conversión clara

### Regla de superficie

- el sistema debe apoyarse en `tonal layering`, cambios de fondo y separación por ritmo visual
- los divisores explícitos, grids con líneas visibles o separadores innecesarios no forman parte de la gramática ideal
- la profundidad visual debe venir de capas, contraste tonal, framing y aire, no de efectos teatrales

## Rol de cada modo

### `dark`

Usar como referencia principal cuando se quiera:

- máxima firma visual de Anclora
- una atmósfera más autoritativa y distintiva
- una lectura más editorial, densa y estratégica
- una landing que deba sentirse más `flagship`

### `light`

Usar como referencia principal cuando se quiera:

- mayor claridad ambiental
- tono premium más abierto y menos dramático
- mejor encaje con productos donde la legibilidad o la neutralidad operativa deban pesar más
- una adaptación más ligera sin salir del mismo sistema

## Bloques canónicos del sistema

La estructura base aprobada queda compuesta por estos módulos:

- `hero`
- `value proposition`
- `narrative / product story`
- `trust / authority`
- `faq`
- `final cta`
- `footer`

Estos bloques deben entenderse como piezas reutilizables del ecosistema, no como secciones cerradas para un único vertical.

## Criterio de reutilización

El sistema debe poder adaptarse a:

- productos AI
- plataformas editoriales
- productos de datos
- servicios premium
- plataformas híbridas del ecosistema

La referencia no debe volver a caer en una estética demasiado específica de `real estate`, `financial advisory` o `portfolio showcase`, salvo que el producto concreto lo requiera de forma explícita.

## Archivo de investigación

La exploración completa de Stitch queda archivada en:

- `research/stitch/`

Dentro de esa carpeta se conserva únicamente:

- los dos diseños finales normalizados
- la cadena de prompts utilizada para llegar al resultado final

Esto deja `resources/` reservado para referencias estables y canónicas.

1. Leer [[ANCLORA_ECOSYSTEM_CONTRACT_GROUPS]].
2. Leer [[ANCLORA_PREMIUM_APP_CONTRACT]].
3. Tomar `dark` como referencia de identidad base.
4. Ajustar `light` como variante de aplicación cuando el caso de uso lo pida.
5. Adaptar copy, módulos e imaginería al producto concreto sin romper la gramática de grupo.
