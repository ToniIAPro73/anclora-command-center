## Notas relacionadas

* Esta nota desarrolla un problema central planteado en [[El Arquitecto del Segundo Cerebro Autónomo]].
* Se relaciona con [[Guía Maestra de Comandos y Hooks para Claude Code]] por el uso de `/resume`, compactación y memoria operativa.
* Puede convertirse en un principio o playbook dentro de `sistemas/` o `playbooks/` si lo adoptas como flujo estable.

El sistema **CPR (Compress, Preserve, Resume)**, documentado en el repositorio de Elia Alberti, es un conjunto de tres habilidades (skills) personalizadas para Claude Code diseñadas específicamente para resolver el problema de la pérdida de contexto y gestionar eficientemente la memoria (y el consumo de tokens) al integrar la IA con bóvedas de Obsidian 1-4. Su objetivo es evitar que el usuario tenga que explicar su proyecto desde cero en cada nueva sesión 3, 5\.  
Para gestionar el límite de tokens en un cerebro digital, estos comandos configuran un flujo de trabajo cíclico:

* **/compress (Guardar sesión):** Se debe utilizar antes de finalizar una sesión de trabajo 6, 7\. El comando permite seleccionar qué información valiosa se desea conservar (aprendizajes clave, decisiones, tareas pendientes o errores) y **crea un registro de sesión con un resumen estructurado** y la conversación completa 6, 7\. Este resumen de "Referencia rápida" está optimizado para que la IA lo escanee en el futuro de manera veloz y con un **bajo uso de tokens** 8, 9\.  
* **/preserve (Actualizar memoria permanente):** Se utiliza para documentar conocimientos a largo plazo que la IA siempre debe saber sobre el proyecto, como convenciones de código, rutas de archivos clave o decisiones de arquitectura, agregándolos al archivo CLAUDE.md 10, 11\. Para evitar que este archivo crezca indefinidamente y consuma todo el límite de tokens, la habilidad cuenta con una **lógica de auto-archivo integrada**: cuando el archivo supera las 280 líneas, identifica y mueve de forma segura el contenido antiguo a un archivo CLAUDE-Archive.md, manteniendo el CLAUDE.md principal ágil, ligero y relevante para la IA 11-13.  
* **/resume (Cargar contexto):** Se ejecuta al iniciar una nueva sesión para recargar el contexto del proyecto de forma eficiente 3, 5\. El comando lee el archivo CLAUDE.md y los resúmenes creados por /compress 14, 15\. Para minimizar aún más el uso de tokens, el comando es consciente del proyecto actual y **permite cargar un contexto filtrado**. Por ejemplo, puedes ejecutar /resume 5 jira para que la IA cargue únicamente las últimas 5 sesiones que mencionen "jira", evitando saturar la ventana de contexto con historiales irrelevantes 14, 15\.

En conjunto, el sistema CPR evita la acumulación infinita de tokens al separar la memoria activa y resumida de los registros históricos profundos, garantizando que el agente de IA siempre tenga el contexto correcto sin sobrecargar su capacidad de procesamiento 8, 12, 14\.
